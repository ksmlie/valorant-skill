# valorant-pro-skill

`valorant-pro-skill` 是一个面向《无畏契约》职业赛事的查询型 Cursor Skill，用于稳定回答选手、战队、比赛、赛程与赛事汇总相关问题，并将结果组织为便于机器解析的结构化数据。

## 主要功能

- 选手信息与近 90 天战绩查询
- 战队信息、当前阵容与近期状态查询
- 比赛结果与交手记录查询
- 赛程、今日比赛与未来比赛查询
- 赛事信息查询：总页 + 全部子页面汇总 + 参赛队伍 / 阶段 / 排名整理

## 数据源原则

- 仅使用 `VLR.gg` 真实页面数据
- 不再使用 `Liquipedia`
- 涉及选手数据时，必须遵循 `references/vlr-data-refresh.md`
- 涉及赛事数据时，必须先读取总页，再查找并汇总所有子页面
- 不得基于单个子页面直接得出完整赛事结论

## 静态数据约定

- `data/teams.json`：战队身份、赛区、别名、当前阵容
- `data/players.json`：选手身份、所属战队、基础信息与 90 天逐英雄统计
- `data/events.json`：赛事信息、子页面、参赛队伍、阶段与排名

## 赛事查询规则

1. 先读取赛事总页，确认赛事基本信息。
2. 查找总页中所有可用的子页面链接，例如 `swiss-stage`、`playoffs`、`group-stage`。
3. 逐个打开每个子页面，读取其中展示的队伍、排名、阶段结果和其他可见信息。
4. 对于已经结束的赛事：
   - 合并所有子页面后再统计参赛队伍。
   - 子页面中展示的排名标号，视为该赛事的最终排名信息。
5. 对于未开始或进行中的赛事：
   - 不关注最终排名。
   - 只统计所有子页面合并后的参赛队伍集合。
   - 如果某些队伍暂未有最终名次，可留空。
6. 永远不要只根据单个子页面就做最终汇总。

## 选手查询规则

- 先查队伍页 `Current Roster`
- 再查选手主页
- 再查 `?timespan=90d`
- 只写真实可见字段，不猜测、不补全
- 选手统计按英雄逐行保存；英雄名缺失时可留空

## 输出要求

- 优先返回结构化 JSON 风格结果
- 使用标准 `id`、别名和可追踪的来源字段
- 找不到精确结果时返回结构化 `not_found`
- 保持字段稳定、内容真实、格式紧凑
